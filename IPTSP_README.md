# IPTSP Manager - Bangladesh & India SIP Trunk Management System

Complete IPTSP (IP Telephony Service Provider) management solution for Bangladesh and India SIP trunk operations with WireGuard tunnel integration.

## 📋 Features

### Core Functionality
- ✅ **Bangladesh SIP Numbers** - User/Password/SIP Server configuration (Port 5080)
- ✅ **India SIP Trunk** - type=peer configuration with SBC integration (Port 5060)
- ✅ **WireGuard VPN** - Secure tunnel between Bangladesh and India
- ✅ **Asterisk PBX** - Full SIP proxy and call routing
- ✅ **Port Scanner** - Monitor UDP/TCP ports for SIP traffic
- ✅ **Carrier Management** - Inbound/Outbound routing per carrier
- ✅ **Web GUI** - Modern React-based admin panel
- ✅ **Authentication** - Secure login with session management

### Country-Specific Configuration

#### 🇧🇩 Bangladesh
| Parameter | Value |
|-----------|-------|
| Config Type | User/Pass/SIP Server |
| SIP Port | 5080 (UDP) |
| WireGuard | 10.100.0.0/24 |
| SIP Server | 10.100.0.1 |
| NAT | Yes |
| Codecs | ulaw, alaw, gsm |
| Context | bangladesh-inbound |

**Note:** Port 5060 is blocked in Bangladesh. Use 5080, 5081, or 5082.

#### 🇮🇳 India
| Parameter | Value |
|-----------|-------|
| Config Type | type=peer |
| SIP Port | 5060 (UDP) |
| Host | 100.64.216.4 |
| From Domain | 100.65.166.6 |
| Default User | +914223532220 |
| Context | from-trunk |
| Insecure | invite,port |
| Codecs | g729, ulaw, alaw |
| NAT | Yes |

**SIP URI:** `sip:+914223532220@100.65.166.6`

**Dialplan Patterns:**
- `X.` - Matches any number
- `0 0091 X.` - International format

## 🚀 Quick Start

### 1. Deploy Asterisk Server

```bash
# Download deployment script
wget https://your-server.com/deploy_iptsp.sh
chmod +x deploy_iptsp.sh

# Run as root
sudo bash deploy_iptsp.sh
```

This script will:
- Install/update Asterisk 20
- Configure WireGuard tunnels (BD & IN)
- Set up iptables/firewall
- Configure all Asterisk configs
- Start all services

### 2. Setup WireGuard Tunnel

**On Bangladesh Server:**
```bash
sudo bash setup_wireguard_tunnel.sh
# Select: 1 (Bangladesh Relay)
# Enter India server's public key
# Enter India server's public IP
```

**On India Server:**
```bash
sudo bash setup_wireguard_tunnel.sh
# Select: 2 (India Tunnel)
# Enter Bangladesh server's public key
# Enter Bangladesh server's public IP
```

### 3. Access Web GUI

```bash
# The web app is a single HTML file
# Open dist/index.html in a browser
# Or serve with nginx/apache

# Default login:
Username: admin
Password: admin123
```

### 4. Configure SIP Numbers

**Bangladesh:**
1. Go to Bangladesh page
2. Click "Add Bangladesh Number"
3. Enter number (e.g., +8801712345678)
4. Set SIP Server (10.100.0.1)
5. Port: 5080
6. Save and copy credentials

**India:**
1. Go to India page
2. Click "Add India Trunk"
3. Configure peer settings
4. Set host: 100.64.216.4
5. From domain: 100.65.166.6
6. Save configuration

## 🔧 Asterisk Configuration Files

### pjsip.conf
Main PJSIP configuration with:
- Transports for BD (5080) and IN (5060)
- Endpoint templates
- Bangladesh SIP number definitions
- India trunk peer configuration
- Registration settings

### extensions.conf
Dialplan with:
- Bangladesh inbound context
- India outbound context
- International routing (0091, +91, 00880, +880)
- Local routing patterns

### sip.conf
Legacy SIP configuration for older devices

### manager.conf
AMI (Asterisk Manager Interface) configuration:
- Port: 5038
- Users: admin, iptsp


## 🛠️ Advanced Configuration

### Custom Codecs
Edit pjsip.conf endpoint templates:
```ini
[endpoint-bangladesh](!)
...
allow=opus
allow=g722
allow=ulaw
allow=alaw
```

### CDR (Call Detail Records)
Enable MySQL CDR:
```bash
# Install MySQL
apt-get install -y mysql-server

# Create database
mysql -e "CREATE DATABASE net2app CHARACTER SET utf8mb4;"
mysql -e "CREATE USER 'net2app'@'localhost' IDENTIFIED BY 'Telco1988';"
mysql -e "GRANT ALL ON net2app.* TO 'net2app'@'localhost';"

# Configure cdr_mysql.conf
# Edit /etc/asterisk/cdr_mysql.conf
```

### Fail2ban for Asterisk
```bash
apt-get install -y fail2ban

cat > /etc/fail2ban/jail.local <<EOF
[asterisk-iptables]
enabled  = true
filter   = asterisk
action   = iptables-allports[name=ASTERISK, protocol=all]
logpath  = /var/log/asterisk/messages
maxretry = 5
bantime  = 3600
EOF

systemctl restart fail2ban
```

## 🆘 Support & Troubleshooting

### Common Issues

**1. SIP Registration Fails**
- Check WireGuard tunnel status
- Verify firewall rules
- Check credentials
- Review Asterisk logs

**2. One-Way Audio**
- Verify NAT settings
- Check RTP port range (10000-20000)
- Ensure symmetric RTP is enabled

**3. WireGuard Tunnel Down**
- Check both server configs
- Verify public keys match
- Test UDP connectivity
- Check iptables rules

**4. High Latency**
- Use recommended ports (scan with Port Scanner)
- Check network path
- Optimize codec selection
